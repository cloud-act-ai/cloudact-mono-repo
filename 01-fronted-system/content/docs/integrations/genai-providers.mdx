## OpenAI Integration

### Prerequisites

Before connecting OpenAI to CloudAct:

1. **OpenAI account** with API access
2. **API key** with usage read permissions
3. **Organization ID** (for organization accounts)
4. **Billing history** (costs appear after first API usage)

### Step 1: Create an API Key

1. Log in to [OpenAI Platform](https://platform.openai.com)
2. Navigate to **API Keys** in the left sidebar
3. Click **Create new secret key**
4. Configure the key:

| Setting | Value |
|---------|-------|
| **Name** | `CloudAct Usage Reader` |
| **Permissions** | Read (usage data) |
| **Project** | All projects (recommended) |

5. Click **Create secret key**
6. **Copy the key immediately** (shown only once)

<Callout type="warning">
  OpenAI API keys are displayed only once when created. Copy and store securely before closing the dialog.
</Callout>

### Step 2: Find Your Organization ID

1. Go to [Organization Settings](https://platform.openai.com/account/organization)
2. Copy the **Organization ID** (starts with `org-`)

<Callout type="info">
  Organization ID is optional for personal accounts but required for organization billing.
</Callout>

### Step 3: Connect in CloudAct

1. Navigate to **Integrations** → **GenAI Providers**
2. Click **Connect OpenAI**
3. Enter credentials:

| Field | Value |
|-------|-------|
| **API Key** | `sk-...` (your secret key) |
| **Organization ID** | `org-...` (optional) |

4. Click **Verify Connection**
5. If successful, click **Save**

## Google AI (Gemini) Integration

### Prerequisites

Before connecting Google AI to CloudAct:

1. **Google Cloud account** or **AI Studio account**
2. **API key** (AI Studio) or **Service account** (Vertex AI)
3. **Gemini API enabled** in your project

### Option A: AI Studio (API Key)

For Google AI Studio access:

#### Step 1: Get API Key

1. Go to [Google AI Studio](https://aistudio.google.com/)
2. Click **Get API Key**
3. Select or create a Google Cloud project
4. Copy the API key

#### Step 2: Connect in CloudAct

1. Navigate to **Integrations** → **GenAI Providers**
2. Click **Connect Google AI**
3. Select **AI Studio (API Key)**
4. Enter:

| Field | Value |
|-------|-------|
| **API Key** | Your AI Studio API key |

5. Click **Verify Connection**
6. Click **Save**

### Option B: Vertex AI (Service Account)

For enterprise Vertex AI access:

#### Step 1: Enable Vertex AI

1. Go to [Google Cloud Console](https://console.cloud.google.com)
2. Navigate to **APIs & Services** → **Enable APIs**
3. Search for and enable:
   - Vertex AI API
   - Generative Language API

#### Step 2: Create Service Account

1. Go to **IAM & Admin** → **Service Accounts**
2. Click **Create Service Account**
3. Configure:

| Field | Value |
|-------|-------|
| **Name** | `cloudact-vertex-reader` |
| **Description** | CloudAct Vertex AI usage reader |

4. Grant roles:

| Role | Purpose |
|------|---------|
| `roles/aiplatform.user` | Access Vertex AI |
| `roles/monitoring.viewer` | View usage metrics |

5. Create and download JSON key

#### Step 3: Connect in CloudAct

1. Navigate to **Integrations** → **GenAI Providers**
2. Click **Connect Google AI**
3. Select **Vertex AI (Service Account)**
4. Enter:

| Field | Value |
|-------|-------|
| **Project ID** | Your GCP project ID |
| **Service Account JSON** | Upload or paste key |

5. Click **Verify Connection**
6. Click **Save**

## Custom Pricing Configuration

If you have negotiated enterprise rates:

### Setting Custom Pricing

1. Go to **Integrations** → **GenAI Providers**
2. Click **Edit Pricing** on the provider
3. Enter your custom rates per model:

| Field | Description |
|-------|-------------|
| **Model** | Select the model |
| **Input Rate** | Your per-token/character rate |
| **Output Rate** | Your per-token/character rate |
| **Effective Date** | When pricing takes effect |

4. Click **Save Custom Pricing**

### Pricing Priority

CloudAct applies pricing in this order:

1. **Custom pricing** (if configured)
2. **Real-time provider pricing** (default)

## Managing Connections

### Connection Status

| Status | Description |
|--------|-------------|
| **Connected** | Active and syncing usage |
| **Error** | Authentication failed |
| **Rate Limited** | Too many requests (auto-retry) |
| **Syncing** | Currently fetching data |

### Updating API Keys

When keys expire or rotate:

1. Click **Edit** on the provider
2. Enter new API key
3. Click **Verify Connection**
4. Click **Save**

### Removing a Connection

1. Click **Remove** on the provider
2. Confirm removal
3. Historical usage data is retained

### Sync Schedule

| Provider | Frequency | Data Delay |
|----------|-----------|------------|
| OpenAI | Every 6 hours | 24 hours |

### Manual Sync

1. Go to **Pipelines** → **GenAI Runs**
2. Select the provider
3. Click **Run Now**
4. Select date range
5. Confirm

<Callout type="info">
  GenAI usage data typically has a 24-hour delay from providers.
</Callout>

## Cost Calculation

### How Costs Are Calculated

```
Cost = (Input Tokens × Input Price) + (Output Tokens × Output Price)
```

### Token Counting

| Provider | Token Unit |
|----------|------------|
| OpenAI | Tokens (GPT tokenizer) |
| Anthropic | Tokens (Claude tokenizer) |
| Google AI | Characters |

### Cost Aggregation

CloudAct aggregates costs by:

- **Daily** totals
- **Model** breakdown
- **Project/Workspace** (if available)

## Troubleshooting

### Invalid API Key

| Symptom | Cause | Solution |
|---------|-------|----------|
| Connection fails | Key expired | Generate new key |
| Connection fails | Key revoked | Generate new key |
| Connection fails | Wrong key format | Verify key prefix |

### No Usage Data

1. **Check API usage** - Ensure you have API calls in the date range
2. **Wait for sync** - Data has 24-hour delay
3. **Verify permissions** - Key must have usage read access
4. **Check pipeline** - View GenAI Runs for errors

### Incorrect Costs

1. **Check pricing** - Verify pricing is current
2. **Custom rates** - Ensure custom pricing is correct
3. **Model mapping** - Verify model names match

## Security Best Practices

1. **Use read-only keys** - Only grant usage read access
2. **Dedicated keys** - Create keys specifically for CloudAct
3. **Regular rotation** - Rotate keys every 90 days
4. **Monitor usage** - Review API key usage in provider consoles
5. **Remove unused** - Delete connections you no longer need

<Callout type="tip">
  Create dedicated API keys with minimal permissions for CloudAct. Never use keys with write or admin access.
</Callout>
